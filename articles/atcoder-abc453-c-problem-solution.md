# 問題概要
座標0.5からスタートし、各移動で正または負方向に進む。
移動の順番は固定だが、方向は自由に選べる。

このとき、座標0を何回通過できるかの最大値を求める問題。

# 制約と考察

N ≤ 20 と小さいため、各移動の方向（正・負）を全探索できる。

各移動で2通りの選択があるため、全体で 2^N 通りを試すことができる。
2^20 ≒ 100万 なので十分間に合う。

したがって、bit全探索で各パターンを試す方針を取る。

# 解法

各移動について「正方向に進むか」「負方向に進むか」をbitで表現する。

bitが1なら正方向、0なら負方向とすることで、全ての移動パターンを列挙できる。

それぞれのパターンについて、実際に移動をシミュレーションし、
座標が正から負、または負から正に変化した回数をカウントする。

この回数が「座標0を通過した回数」に対応する。

初期位置は 0.5 だが、実際には「正か負か」だけを判定すればよい。

したがって、初期値を 0 として扱い、
符号の変化のみを追跡すれば問題ない。

全てのパターンの中で最大値を答えとする。

# 実装
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main()
{
    int n;
    cin >> n;

    int ans = 0;
    vector<int> l(n);
    for(int i = 0; i < n; i++) cin >> l[i];
    for(int bit = 0; bit < (1<<n); bit++)
    {
        long long now = 0;
        int cnt = 0;
        for(int i = 0; i < n; i++)
        {
            bool p = now >= 0;
            if(bit & (1<<i)) now += l[i];
            else now -= l[i];
            if(p && now < 0) cnt++;
            else if(!p && now >= 0) cnt++;
        }
        ans = max(ans, cnt);
    }
    cout << ans << endl;
}
```

# まとめ

Nが小さい場合は、bit全探索を考えるのが有効。

また、今回のように「符号の変化」に着目することで、
問題をシンプルに扱うことができる。
